# API Curl Examples

Ready-to-run `curl` examples for every endpoint.

Base URL (local run): `http://127.0.0.1:8000`
If you run via Docker, you can also use `http://localhost:8000`.

## Health

```bash
curl "http://127.0.0.1:8000/health"
```

## Ask

```bash
curl -X POST "http://127.0.0.1:8000/ask" \
  -H "Content-Type: application/json" \
  -d '{"question":"Explain async/await in Python in simple words."}'
```

## Ask Stream

```bash
curl -N -X POST "http://127.0.0.1:8000/ask-stream" \
  -H "Content-Type: application/json" \
  -d '{"question":"Write a short paragraph about FastAPI streaming."}'
```

## Tool Assistant

Lets the model choose safe backend tools: `get_weather`, `search_docs`,
`create_ticket`, and `send_email_draft`.

```bash
curl -X POST "http://127.0.0.1:8000/tool-assistant" \
  -H "Content-Type: application/json" \
  -d '{"message":"What is the weather in Yerevan, and create an email draft to tell the team?"}'
```

> Full explanation: `doc/5. TOOL_CALLING.md`.

## Upload document (.txt or .pdf)

```bash
curl -X POST "http://127.0.0.1:8000/documents/upload" \
  -F "file=@./sample.txt"
```

```bash
curl -X POST "http://127.0.0.1:8000/documents/upload" \
  -F "file=@./handbook.pdf"
```

Example response:

```json
{
  "filename": "sample.txt",
  "chunks_stored": 4,
  "total_characters": 1672
}
```

## Semantic search

```bash
curl -X POST "http://127.0.0.1:8000/documents/search" \
  -H "Content-Type: application/json" \
  -d '{"query":"How does retry logic work?","top_k":3}'
```

Example response:

```json
{
  "query": "How does retry logic work?",
  "results": [
    {
      "text": "Retries use exponential backoff to reduce pressure...",
      "filename": "sample.txt",
      "similarity": 0.9241
    },
    {
      "text": "Timeout handling returns a friendly API message...",
      "filename": "sample.txt",
      "similarity": 0.8819
    }
  ]
}
```

> For advanced search options (`filename`, `keyword`, `min_similarity`),
> see `doc/2. VECTOR_DATABASE_USAGE.md`.

## Chat with documents (RAG, with citations)

Generates an answer grounded only in your uploaded documents.

```bash
curl -X POST "http://127.0.0.1:8000/documents/ask" \
  -H "Content-Type: application/json" \
  -d '{"question":"How does retry logic work?"}'
```

Optional fields: `top_k`, `filename`, `min_similarity`.

```bash
curl -X POST "http://127.0.0.1:8000/documents/ask" \
  -H "Content-Type: application/json" \
  -d '{"question":"How does retry logic work?","top_k":4,"filename":"sample.txt","min_similarity":0.2}'
```

Example response:

```json
{
  "question": "How does retry logic work?",
  "answer": "Retries use exponential backoff [1], and timeouts return a friendly message [2].",
  "used_context": true,
  "citations": [
    { "index": 1, "filename": "sample.txt", "text": "Retries use exponential backoff...", "similarity": 0.9241 },
    { "index": 2, "filename": "sample.txt", "text": "Timeout handling returns a friendly API message...", "similarity": 0.8819 }
  ],
  "model": "gpt-4o-mini",
  "tokens_used": 320
}
```

When nothing relevant is found, the API abstains instead of guessing:

```json
{
  "question": "What is the capital of France?",
  "answer": "I don't know based on the uploaded documents. I couldn't find anything relevant to that question.",
  "used_context": false,
  "citations": [],
  "model": null,
  "tokens_used": 0
}
```
